# Jekyll Automation Guide

This site is now fully automated with Jekyll. Here's how everything works:

## Automatic Features

### 1. Sitemap Generation (No Manual Work!)
- **Plugin**: `jekyll-sitemap` in `_config.yml`
- **What it does**: Automatically generates `sitemap.xml` from all your pages and posts
- **You do**: Nothing! Just add posts and Jekyll handles the rest
- **Location**: Auto-generated at `/sitemap.xml`

### 2. RSS Feed (Bonus!)
- **Plugin**: `jekyll-feed` in `_config.yml`
- **What it does**: Auto-generates RSS feed for your blog
- **Location**: Auto-generated at `/feed.xml`

### 3. Related Posts (Smart Matching!)
- **Location**: Built into `_layouts/post.html`
- **How it works**:
  - Automatically finds posts with same category
  - Matches posts with similar tags
  - Falls back to recent posts if no matches
- **You do**: Just add `category` and `tags` to your frontmatter

### 4. Automatic Article Listings
- **Include**: `_includes/posts-by-category.html`
- **What it does**: Automatically groups and displays posts by category
- **You do**: Nothing! Just categorize your posts correctly

## How to Add a New Article

### Step 1: Create the File
Create a markdown file in the root directory with this naming format:
```
article-name.md
```

### Step 2: Add Front Matter
At the top of your file, add this (copy and modify):

```yaml
---
layout: post
title: "Your Article Title Here"
description: "Your article description (150-160 chars for SEO)"
keywords: "keyword1, keyword2, keyword3"
author: "Generalist Programmer"
category: "Technical Guides"  # Options: "Technical Guides", "Case Studies", "Product Guides"
tags: ["tag1", "tag2", "tag3"]  # Used for related posts matching
date: 2025-11-18
read_time: 10
modified: 2025-11-18
canonical_url: "https://generalistprogrammer.github.io/your-article-name.html"
robots: "index, follow"
---
```

### Step 3: Write Your Content
Just write your markdown content below the front matter. That's it!

### Step 4: Push to GitHub
```bash
git add .
git commit -m "Add new article"
git push
```

Jekyll automatically:
- ✅ Adds it to the sitemap
- ✅ Generates related posts based on category/tags
- ✅ Lists it on homepage by category
- ✅ Creates RSS feed entry
- ✅ Generates proper canonical URLs
- ✅ Adds SEO meta tags

## Categories Explained

Use these exact category names in your frontmatter:

- **"Product Guides"**: Guides about specific products/services
- **"Case Studies"**: Real-world project experiences
- **"Technical Guides"**: Educational technical content

## Tags for Better Related Posts

Add relevant tags to help Jekyll match related content:

```yaml
tags: ["startup", "saas", "automation", "python", "api"]
```

Articles with matching tags and categories will automatically appear as related posts!

## Advanced: Using Includes

You can use the custom includes anywhere:

### Show all posts by category:
```liquid
{% include posts-by-category.html %}
```

### Show article grid:
```liquid
{% include article-grid.html category="Case Studies" limit=5 %}
```

### Show related posts:
```liquid
{% include related-posts.html %}
```

## Troubleshooting

**Sitemap not updating?**
- Make sure `jekyll-sitemap` is in `_config.yml` plugins
- Check that your files have proper frontmatter

**Related posts not showing?**
- Add `category` and `tags` to your frontmatter
- Make sure you're using the `post` layout

**Article not appearing on homepage?**
- Verify the `category` field matches exactly: "Product Guides", "Case Studies", or "Technical Guides"
- Check that `layout: post` is in frontmatter
